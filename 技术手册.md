## 📄 文件详细说明

### 🔧 核心配置文件

| 文件 | 作用 | 关键内容 |
|------|------|----------|
| docker-compose.yml | 定义4个Docker服务 | backend(API)、frontend(网页)、mqtt(消息)、nginx(代理) |
| deploy.sh | 一键部署脚本 | 自动创建目录、构建镜像、启动服务 |
| .env.example | 环境变量模板 | PORT、DB路径、MQTT地址、JWT密钥 |

### 📦 后端 (backend/)

| 文件 | 作用 | 代码行数 |
|------|------|----------|
| server.js | **主服务入口** | 415行 |
| database.js | 数据库操作层 | 184行 |
| package.json | 依赖声明 | - |

#### server.js 核心功能：

```
┌─────────────────────────────────────────────────────┐
│                   server.js 功能模块                 │
├─────────────────────────────────────────────────────┤
│ 1. Express HTTP服务 (端口3000)                        │
│    - /api/auth/login     用户登录                    │
│    - /api/devices        设备CRUD                   │
│    - /api/system/status  系统状态                    │
│    - /api/health         健康检查                   │
│                                                     │
│ 2. WebSocket服务 (路径/ws)                           │
│    - 实时推送设备状态                                │
│    - 接收设备数据                                    │
│    - 发送控制命令                                    │
│                                                     │
│ 3. MQTT Broker客户端                                │
│    - 订阅: device/+/status/data/heartbeat           │
│    - 发布: device/ID/command                        │
│                                                     │
│ 4. JWT认证中间件                                     │
│ 5. 限流保护 (15分钟100次)                           │
└─────────────────────────────────────────────────────┘
```

#### database.js 数据库表：

| 表名 | 用途 | 主要字段 |
|------|------|----------|
| devices | 设备信息 | device_id, name, type, status, ip, last_seen |
| device_data | 设备数据历史 | device_id, data_type, data_value |
| users | 用户账户 | username, password(加密), role |
| command_logs | 命令执行记录 | device_id, command, params |

### 🎨 前端 (frontend/)

#### 路由页面 (router/index.js)

| 路径 | 组件 | 说明 |
|------|------|------|
| /login | Login.vue | 登录页面 |
| /dashboard | Dashboard.vue | 控制台首页 |
| /devices | Devices.vue | 设备列表 |
| /devices/:id | DeviceDetail.vue | 设备详情 |
| /bluetooth | Bluetooth.vue | 蓝牙配网 |
| /settings | Settings.vue | 系统设置 |

#### 状态管理 (stores/)

**user.js - 用户状态**

```
Token存储 → localStorage → axios请求头
登录/登出 → Token管理 → 路由守卫
```

**websocket.js - WebSocket管理**

```
连接 → 订阅设备 → 接收消息 → 状态更新
自动重连 (最多5次，间隔递增)
```

#### 页面功能

| 页面 | 功能 |
|------|------|
| Login.vue | 用户名密码登录，JWT Token获取 |
| Dashboard.vue | 统计卡片、在线设备列表、系统状态 |
| Devices.vue | 设备增删改查、发送控制命令 |

## 🚀 支持的部署平台

### 1. NAS（群晖、威联通等）✅ 推荐
- 配置文件 docker-compose.yml 就是为NAS优化的
- 支持群晖DSM、威联通QTS等主流NAS系统

### 2. Linux服务器
- Ubuntu / Debian / CentOS / Rocky Linux
- 需要安装Docker和Docker Compose

### 3. Windows
- 通过 WSL2（Windows Subsystem for Linux）
- 或 Docker Desktop for Windows

### 4. 云服务器
- 阿里云、腾讯云、AWS等
- 只要能运行Docker即可

### 5. 树莓派 / SBC单板机
- Raspberry Pi OS
- Arm64架构支持

### 部署前提

```bash
# 需要安装
- Docker
- Docker Compose
```

## 🚀 快速部署

```bash
# 1. 克隆到NAS
cd /volume1/docker
git clone <项目> iot-platform
cd iot-platform

# 2. 一键部署
chmod +x deploy.sh
./deploy.sh

# 3. 访问
http://NAS_IP          # 前端界面
http://NAS_IP:3000     # API服务
tcp://NAS_IP:1883      # MQTT

# 4. 登录
用户名: admin
密码: admin123
```

## 📝 MQTT主题约定

| 主题 | 方向 | 用途 |
|------|------|------|
| device/ID/status | 设备→服务器 | 设备状态上报 |
| device/ID/data | 设备→服务器 | 传感器数据 |
| device/ID/heartbeat | 设备→服务器 | 心跳保活 |
| device/ID/command | 服务器→设备 | 下发命令 |
| device/ID/config/wifi | 服务器→设备 | WiFi配置 |

## 📊 技术栈

- **后端**: Node.js + Express + SQLite + MQTT
- **前端**: Vue3 + Element Plus + ECharts + Pinia
- **设备**: ESP32 + Arduino + NimBLE
- **部署**: Docker + Docker Compose
- **消息**: Mosquitto MQTT Broker
- **代理**: Nginx反向代理
