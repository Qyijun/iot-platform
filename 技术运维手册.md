## 部署流程

### 1. 基础环境安装

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install -y nodejs npm git

# 安装 PM2
sudo npm install -g pm2

# 安装 nodemon（开发环境热重载）
sudo npm install -g nodemon
```

### 2. 项目拉取

```bash
# 创建目录
mkdir -p /opt/iot-platform
cd /opt/iot-platform

# 克隆代码
git clone https://github.com/Qyijun/iot-platform.git .

# 如果拉取超时，配置 Git HTTP
git config --global http.version HTTP/1.1
```

### 3. 后端部署

```bash
cd backend

# 安装依赖（使用国内镜像）
npm install --registry https://registry.npmmirror.com

# 配置环境变量
cp ../.env.example .env.local
# 编辑 .env.local 设置正确的配置

# 启动服务
pm2 start server.js --name iot-backend

# 保存 PM2 配置（开机自启）
pm2 save
pm2 startup
```

### 4. 前端构建

```bash
cd ../frontend

# 安装依赖
npm install --registry https://registry.npmmirror.com

# 构建生产版本
npm run build
```

### 5. 防火墙设置

```bash
# 开放端口
sudo ufw allow 3000/tcp  # 后端API
sudo ufw allow 1883/tcp  # MQTT（可选）
sudo ufw enable
```

## 日常运维命令

### PM2 进程管理

```bash
# 查看进程状态
pm2 list
pm2 status

# 重启服务
pm2 restart iot-backend

# 停止服务
pm2 stop iot-backend

# 查看日志

pm2 logs iot-backend --f

pm2 delete iot-backend

pm2 info iot-backend
```

```bash
cd /opt/iot-platform

# 拉取最新代码
git pull

# 进入后端目录更新
cd backend
pm2 restart iot-backend

# 进入前端目录重新构建
cd ../frontend
npm run build
```

## 常见问题排查

### 1. 服务无法启动

**错误**: `ERR_CONNECTION_REFUSED` 或 `无法访问此网站`

**排查步骤**:

```bash
# 1. 检查进程是否运行
pm2 list

# 2. 如果没有运行，手动启动查看错误
cd /opt/iot-platform/backend
node server.js
```

**常见原因**:

| 错误信息 | 原因 | 解决方案 |
|----------|------|----------|
| `SyntaxError` | server.js 代码有语法错误 | `git pull` 拉取最新代码 |
| `MODULE_NOT_FOUND` | 依赖未安装 | `npm install` |
| `EADDRINUSE` | 端口被占用 | `pm2 stop all` 或修改端口 |
| `这个` + 代码 | 文件开头有乱码 | 删除乱码字符 |

**修复后重启**:

```bash
pm2 restart iot-backend
```

---

### 2. 前端白屏

**错误**: 页面加载后显示空白，控制台有 `ERR_SSL_PROTOCOL_ERROR`

**排查步骤**:

1. 检查浏览器控制台（F12 → Network）
2. 查看是否有 HTTPS 请求

**解决方案**:

修改 `backend/server.js` 中的 helmet 配置：

```javascript
// 禁用 HSTS（允许 HTTP 访问）
app.use(helmet({
  hsts: false,
  contentSecurityPolicy: false
}));
```

**CORS 配置优化**:

```javascript
app.use(cors({
  origin: true,
  credentials: true,
  methods: ['GET', 'POST', 'PUT', 'DELETE', 'OPTIONS', 'PATCH'],
  allowedHeaders: ['Content-Type', 'Authorization']
}));
```

---

### 3. API 请求失败

**错误**: `Failed to load resource: net::ERR_CONNECTION_REFUSED`

**排查步骤**:

```bash
# 1. 检查后端是否运行
curl http://localhost:3000/api/health

# 2. 检查端口监听
netstat -tlnp | grep 3000
# 或
ss -tlnp | grep 3000
```

**解决方案**:

```bash
# 重启后端
pm2 restart iot-backend

# 检查防火墙
sudo ufw status
sudo ufw allow 3000/tcp
```

---

### 4. WebSocket 连接失败

**错误**: `WebSocket connection failed`

**排查步骤**:

```bash
# 1. 检查 WebSocket 服务
curl -i -N -H "Connection: Upgrade" -H "Upgrade: websocket" http://localhost:3000/ws

# 2. 查看后端日志
pm2 logs iot-backend --lines 50
```

---

### 5. MQTT 连接问题

**错误**: `MQTT connection refused` 或 `MQTT connect timeout`

**排查步骤**:

```bash
# 1. 检查 MQTT 服务
netstat -tlnp | grep 1883

# 2. 测试 MQTT 连接
mosquitto_pub -h localhost -p 1883 -t "test" -m "hello"
```

**解决方案**:

检查 `.env.local` 中的 MQTT 配置：

```bash
# MQTT 配置示例
MQTT_BROKER=mqtt://192.168.1.3:1883
MQTT_USERNAME=qiuyijun
MQTT_PASSWORD=Qiuyijun.13825401705
```

---

### 6. npm install 超时

**错误**: `ETIMEDOUT` 或 `Connection timeout`

**解决方案**:

```bash
# 使用国内镜像
npm config set registry https://registry.npmmirror.com

# 或者淘宝镜像
npm config set registry https://registry.np.taobao.org

# 重新安装
npm install
```

## 备份与恢复

### 自动备份脚本

创建 `/opt/iot-platform/backup.sh`:

```bash
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR=/opt/backup/iot-platform
mkdir -p $BACKUP_DIR

# 备份数据库
cp /opt/iot-platform/data/database.sqlite $BACKUP_DIR/database_$DATE.sqlite

# 备份配置
cp /opt/iot-platform/backend/.env.local $BACKUP_DIR/.env.local_$DATE

# 保留最近 30 天备份
find $BACKUP_DIR -mtime +30 -delete

echo "备份完成: $DATE"
```

### 恢复数据

```bash
# 停止服务
pm2 stop iot-backend

# 恢复数据库
cp /opt/backup/iot-platform/database_20240101.sqlite /opt/iot-platform/data/database.sqlite

# 重启服务
pm2 restart iot-backend
```

### 定时备份（crontab）

```bash
# 编辑 crontab
crontab -e

# 添加每天凌晨 3 点备份
0 3 * * * /opt/iot-platform/backup.sh >> /var/log/backup.log 2>&1
```
